# 技术栈说明

## 项目概述

**项目名称**：TeamAgentX 产品讨论论坛  
**项目类型**：前后端分离的 Web 应用  
**开发模式**：单体开发，前后端代码在同一仓库不同目录

---

| 技术 | 版本 | 用途 | 选择理由 |
|------|------|------|----------|

## 技术选型说明

### 为什么使用模板仓库技术栈

本项目直接使用模板仓库 `easy-coding` 的技术栈，原因如下：

1. **统一标准**：所有项目使用相同技术栈，降低学习成本，便于维护
2. **模板成熟**：模板仓库已包含完整配置、约束文件、开发规范
3. **快速启动**：无需从零搭建，直接克隆后定制即可

### 项目专属选型

| 功能 | 选型 | 说明 |
|------|------|------|
| 富文本编辑器 | TipTap | PRD 要求支持富文本编辑（图片、代码块、链接、表格），TipTap 满足需求且轻量 |
| 邮件发送 | nodemailer | PRD 要求邮箱验证，使用 SMTP 发送验证邮件 |
| 频率限制 | 内存缓存 | 发帖、评论频率限制使用内存 Map 记录，简单高效 |

---

## 技术栈对比（不使用的技术）

| 技术 | 为什么不使用 |
|------|--------------|
| Next.js | 前后端分离架构，不需要 SSR |
| Redux | 状态管理过于复杂，Zustand 更轻量 |
| GraphQL | RESTful API 足够，GraphQL 学习成本高 |
| MongoDB | 论坛业务有复杂关系，MySQL 关系型数据库更适合 |
| Redis | 初期简化，频率限制使用内存缓存，后续迭代可引入 |

## 部署方案

**部署方式**：源码部署  
**部署流程**：
1. 本地构建前端和后端代码
2. 上传构建产物到服务器
3. 使用 PM2 或 systemd 运行后端服务
4. 使用 Nginx 或直接监听端口提供前端静态文件

**端口说明**：不使用 80 端口，具体端口由运维配置

## 项目特定约束

### 富文本内容限制
- 帖子内容最多 50000 字符
- 评论内容最多 5000 字符
- 图片最大 5MB，支持 jpg/png/gif

### 业务规则限制
- 每日发帖上限：20 篇/用户
- 评论频率限制：10 条/分钟/用户
- 楼中楼嵌套上限：3 层
- 版块置顶上限：5 篇/版块

### 数据索引
已在数据模型设计文档中定义关键索引，确保查询性能
